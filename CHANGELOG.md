## v0.5.0 (2026-05-07)

### Fix

- **_fast_read**: plug list refcount leak in decode_strings

### Perf

- **types**: add C decode_strings, swap String.read fast path
- **types**: add C decode_datetime, swap DateTime.read fast path
- **types**: add Date/Date32/DateTime read micro-benches and reach the pure-Python ceiling

## v0.4.1 (2026-05-07)

### Fix

- **protocol**: don't loop reading frames in _refill_compressed

### Perf

- **insert**: bulk-pack column writes and skip async-yield per row

## v0.4.0 (2026-05-07)

### Feat

- **perf**: add pytest-benchmark microbenchmark suite under tests/perf/
- **benchmarks**: head-to-head suite vs asynch / clickhouse-connect

### Perf

- **types**: bulk-unpack hot loops, inline String varuint
- **protocol**: make codec read interface synchronous

### Note

A short-lived Rust acceleration experiment (PyO3 `decode_strings`
swap + maturin build backend) landed and was reverted in the same
release window — only the pure-Python sync-codec refactor and the
bulk-unpack pass remain. See `git log` between `v0.3.3` and
`v0.4.0` for the full history.
